# 会话同步记录（精简版）

更新时间：2026-03-06

## 1. 当前项目状态
- `v1_threadpool`：已完成并发文件服务器（线程池版），`list/get/put/quit` 可用。
- `v2_epoll`：已创建并实现 `epoll` 单进程事件循环版本（同级新目录）。
- 运行环境约束：目标是 Linux（板子 AArch64 + 客户端 x86_64/Linux）。

## 2. 已落地的关键实现
- 协议头固定为 `msg_header_t(12B)`，字段：`magic/cmd/status/length`。
- 字节序统一：`htonl/htons/ntohl/ntohs`。
- `v1` 与 `v2` 都兼容命令：`list/get/put/quit`。
- `PUT` 约定：两段式。
  1) `CMD_PUT` + 文件名 payload。
  2) `CMD_PUT` + 文件数据 payload（`length=文件大小`）。
  3) 服务端返回 `CMD_PUT status=0`。

## 6. 换机后最短继续路径
1. `cd <repo>/v2_epoll`
2. `make clean && make server_host && make client_host`
3. 服务端：`./server/server_host 0.0.0.0 9000`
4. 客户端：`./client/client_host <server_ip> 9000`
5. 压测：`./scripts/stress_test.sh <server_ip> 9000 8 5 128`
